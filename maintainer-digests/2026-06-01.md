### Implementation plan — Deterministic Response Cache (for a future pass)
Add an opt-in content-addressed cache keyed by `sha256(provider + model + normalized_prompt + temperature + max_tokens + schema_id)`, persisted in a new `response_cache` table in the existing aiosqlite artifact store (`storage/artifacts.py`). Gate it behind a `--cache/--no-cache` CLI flag and a `defaults.enable_cache` config key, **defaulting to OFF** so adversarial debate keeps its intended variation; only enable by default for `temperature == 0` deterministic calls. On cache hit, short-circuit the provider call in `orchestrator.py` before `_call_provider`, recording a `cache_hit` flag in run metrics so `--verbose`/`--json` surface hit-rate. Add a `council cache clear|stats` subcommand and a configurable TTL (default 7d). Tests: hit/miss/expiry, temperature-0-only auto-enable, and a regression proving non-determinism is preserved when caching is off. Scope it to ~1 focused PR once CI is green; do **not** combine with the security/CI remediation.
